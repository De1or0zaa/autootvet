# Discord Автоответчик — Дизайн-документ

## Обзор

Автоответчик для Discord, работающий от личного токена пользователя. Отвечает случайным сообщением из списка на упоминания (@mention) и ответы (reply) от определённых пользователей на определённом сервере и канале.

## Стек

- **Язык:** Python 3.10+
- **Библиотека:** discord.py-self
- **Конфигурация:** JSON файл (ручное редактирование)
- **Логирование:** Консоль + файл (logs/bot.log)

## Структура проекта

```
автоответчик/
├── config.json          # Токен + правила автоответов
├── main.py              # Точка входа, запуск клиента
├── handlers.py          # Обработчик on_message
├── config_loader.py     # Загрузка/валидация config.json
├── logger_setup.py      # Настройка логирования (консоль + файл)
├── requirements.txt     # Зависимости
└── logs/
    └── bot.log          # Файл логов (создаётся автоматически)
```

## Формат config.json

```json
{
  "token": "ВАШ_ТОКЕН",
  "rules": [
    {
      "guild_id": 123456789,
      "channel_id": 987654321,
      "user_ids": [111111111, 222222222],
      "trigger_on_mention": true,
      "trigger_on_reply": true,
      "responses": [
        "Меня сейчас нет, напишите позже.",
        "Я занят, отвечу позже!",
        "Автоответ: скоро буду."
      ]
    }
  ]
}
```

### Поля правила

| Поле | Тип | Описание |
|------|-----|----------|
| `guild_id` | int | ID сервера Discord |
| `channel_id` | int | ID канала |
| `user_ids` | list[int] | Список ID пользователей, на которых реагировать |
| `trigger_on_mention` | bool | Реагировать на @mention владельца токена |
| `trigger_on_reply` | bool | Реагировать на reply сообщения владельца токена |
| `responses` | list[str] | Список возможных ответов (выбирается случайно) |

## Логика on_message

1. Игнорировать собственные сообщения
2. Игнорировать сообщения от ботов
3. Для каждого правила из конфига:
   - Проверить: `message.guild.id == rule.guild_id`
   - Проверить: `message.channel.id == rule.channel_id`
   - Проверить: `message.author.id in rule.user_ids`
   - Проверить триггер:
     - `trigger_on_mention`: владелец токена упомянут в сообщении
     - `trigger_on_reply`: сообщение является reply на сообщение владельца токена
   - Если все проверки пройдены — отправить случайный ответ из `responses`
4. Логировать каждое срабатывание

## Обработка ошибок

| Ситуация | Действие |
|----------|----------|
| Невалидный токен | Выход с ошибкой в лог |
| Отсутствует config.json | Создать шаблон с примерами, выход |
| Невалидный JSON | Выход с описанием ошибки |
| Ошибка отправки сообщения | Логирование ошибки, продолжение работы |
| Пустой список responses | Пропустить правило, warning в лог |

## Логирование

- **Формат:** `[YYYY-MM-DD HH:MM:SS] [LEVEL] message`
- **Уровни:** INFO (срабатывания), WARNING (некритичные проблемы), ERROR (ошибки)
- **Консоль:** все уровни
- **Файл:** все уровни, ротация при 5MB
